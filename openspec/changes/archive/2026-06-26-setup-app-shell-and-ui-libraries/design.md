# Design: setup-app-shell-and-ui-libraries

## Overview

Bootstrap the app shell by wiping Next.js demo content, installing all core dependencies, configuring shadcn/ui with a concert-brand dark-only theme, and wiring up the root layout with required providers.

---

## 1. Dependency Installation

Install with Bun:

```bash
# Runtime deps
bun add @clerk/nextjs @tanstack/react-query react-hook-form zod

# shadcn/ui (CLI + radix primitives installed on demand via shadcn add)
bunx shadcn@latest init
```

shadcn CLI `init` will add `class-variance-authority`, `clsx`, `tailwind-merge`, and `lucide-react` automatically. Accept all defaults but override the colour theme (see section 3).

## 3. shadcn Theme — Dark-Only Concert Palette

Because the app is dark-only, only one `:root` block is needed (no `.dark` variant toggle). The `<html>` element gets the `dark` class statically in the root layout so shadcn component variants that use `dark:` still resolve correctly.

### Colour tokens (`app/globals.css`)

```css
@import "tailwindcss";
@import "tw-animate-css";

@custom-variant dark (&:is(.dark *));

@theme inline {
  /* Fonts */
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);

  /* Map shadcn CSS vars → Tailwind */
  --color-background:       var(--background);
  --color-foreground:       var(--foreground);
  --color-card:             var(--card);
  --color-card-foreground:  var(--card-foreground);
  --color-popover:          var(--popover);
  --color-popover-foreground: var(--popover-foreground);
  --color-primary:          var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --color-secondary:        var(--secondary);
  --color-secondary-foreground: var(--secondary-foreground);
  --color-muted:            var(--muted);
  --color-muted-foreground: var(--muted-foreground);
  --color-accent:           var(--accent);
  --color-accent-foreground: var(--accent-foreground);
  --color-destructive:      var(--destructive);
  --color-border:           var(--border);
  --color-input:            var(--input);
  --color-ring:             var(--ring);
  --radius-sm:   calc(var(--radius) - 4px);
  --radius-md:   calc(var(--radius) - 2px);
  --radius-lg:   var(--radius);
  --radius-xl:   calc(var(--radius) + 4px);
}

/* Concert dark theme — single theme, no light mode */
:root {
  --radius: 0.625rem;

  /* Deep midnight base */
  --background:         oklch(0.10 0.02 275);
  --foreground:         oklch(0.96 0.005 275);

  /* Card: slightly elevated surface */
  --card:               oklch(0.14 0.025 275);
  --card-foreground:    oklch(0.96 0.005 275);

  /* Popover: same as card */
  --popover:            oklch(0.14 0.025 275);
  --popover-foreground: oklch(0.96 0.005 275);

  /* Primary: electric violet */
  --primary:            oklch(0.62 0.22 295);
  --primary-foreground: oklch(0.99 0 0);

  /* Secondary: subtle indigo surface */
  --secondary:          oklch(0.20 0.04 285);
  --secondary-foreground: oklch(0.92 0.01 280);

  /* Muted: dimmed text and surfaces */
  --muted:              oklch(0.18 0.02 275);
  --muted-foreground:   oklch(0.60 0.015 280);

  /* Accent: neon cyan spark */
  --accent:             oklch(0.72 0.18 210);
  --accent-foreground:  oklch(0.10 0.02 275);

  /* Destructive */
  --destructive:        oklch(0.58 0.22 25);

  /* Borders and inputs */
  --border:             oklch(0.25 0.03 280);
  --input:              oklch(0.22 0.03 280);
  --ring:               oklch(0.62 0.22 295);
}

/* Force dark everywhere */
html {
  color-scheme: dark;
}

body {
  background: var(--background);
  color: var(--foreground);
}
```

## 4. Root Layout

`app/layout.tsx` becomes:

```tsx
import type { Metadata } from "next";
import { Geist, Geist_Mono } from "next/font/google";
import { ClerkProvider } from "@clerk/nextjs";
import Providers from "@/components/providers";
import "./globals.css";

const geistSans = Geist({ variable: "--font-geist-sans", subsets: ["latin"] });
const geistMono = Geist_Mono({ variable: "--font-geist-mono", subsets: ["latin"] });

export const metadata: Metadata = {
  title: "ConcertSeats",
  description: "Find and book the best seats at live concerts",
};

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <ClerkProvider>
      <html
        lang="en"
        className={`${geistSans.variable} ${geistMono.variable} dark h-full antialiased`}
      >
        <body className="min-h-full flex flex-col bg-background text-foreground">
          <Providers>{children}</Providers>
        </body>
      </html>
    </ClerkProvider>
  );
}
```

`ClerkProvider` must wrap the entire tree so `auth()` and `currentUser()` work in Server Components.

### `components/providers.tsx`

Client boundary for TanStack Query:

```tsx
"use client";

import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
import { useState } from "react";

export default function Providers({ children }: { children: React.ReactNode }) {
  const [queryClient] = useState(() => new QueryClient());
  return <QueryClientProvider client={queryClient}>{children}</QueryClientProvider>;
}
```

## Decisions & Trade-offs

| Decision | Rationale |
|---|---|
| Static `dark` class on `<html>` | Simpler than `next-themes`; app never needs a light mode |
| `ClerkProvider` in `layout.tsx`, not `providers.tsx` | Clerk needs to be the outermost wrapper; it's a Server Component itself |
| `Providers` as a separate client component | Keeps `layout.tsx` as a Server Component |
| oklch colour space | Better perceptual uniformity for custom palettes; Tailwind v4 default |
